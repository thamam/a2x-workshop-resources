Workshop flow

How to use this hub live

1. Prepare a safe sandbox.

Use the pre-workshop checklist to bring a toy repo, sanitized sample, starter rules, and verification signal.

2. Choose the right next resource.

Open the route chooser if attendees need a fast map from workshop moment to handout.

3. Route setup confusion safely.

If Claude Code login, token, environment, or policy questions appear, use the credential-routing Q&A before debugging prompts.

4. Open with the harness idea.

Show that better agent results come from environment, rules, tools, feedback loops, and review gates.

5. Give attendees rules they can use today.

Send them to the CLAUDE.md cheat sheet before live coding.

6. Turn repetition into infrastructure.

Use the first-skill guide when moving from tips to reusable workflows, then run Skill Wizard to inspect whether Claude can actually discover and use the drafted skill.

7. Hand out practical prompt tools.

Use the product brief generator, interviewer, prompt improver, and PRD starter when attendees start shaping their own ideas; fall back to the synthetic meeting-prep scenario if they need a safe demo input.

8. Close with one next workflow.

Send attendees to the post-workshop action plan so they leave with a small reversible Claude Code project, not just a pile of links.

9. Package the next agent run.

Use the handoff packet to capture safe inputs, stop conditions, and evidence before the next Claude Code session starts.

10. Score the next trial.

Use the AI tool trial scorecard when attendees need to choose which workflow deserves the next small experiment.

11. Log pilot evidence.

Collect value, friction, risk, reviewer decisions, and open questions before asking a team to continue, expand, or stop.

12. Write the pilot decision memo.

Turn the scored trial into a continue, clarify, or stop recommendation with evidence, boundaries, and one reversible next step.

13. Register pilot risks.

Name the boundary, mitigation, owner, review signal, and stop condition before any pilot expands beyond a reversible test.

14. Send a stakeholder update.

Share the pilot status, evidence, top risk, ask, and next review date without forwarding sensitive examples.

15. Run the pilot retrospective.

Turn pilot evidence into lessons, guardrail updates, honest stop/continue decisions, and one next safe experiment.

16. Check rollout readiness.

Confirm evidence, users, allowed inputs, review gates, rollback path, and stakeholder messaging before the pilot expands.

17. Prepare operations handoff.

Name the owner, support route, monitoring signal, incident trigger, rollback owner, and improvement cadence before routine use.

18. Park advanced demos safely.

Use placeholders for tools until public review is complete.
