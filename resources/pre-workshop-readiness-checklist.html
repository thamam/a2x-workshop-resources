Safe preparation

Bring a tiny sandbox and sanitized examples.

The best workshop target is real enough to matter and small enough to inspect. Do not bring production credentials, customer data, private notes, payment flows, or anything that would be painful if an agent misunderstood the task.

Before the room

Five checks to do ahead of time

1 · Access

Confirm Claude Code opens locally.

Run the tool in a non-sensitive folder and make sure you can start a session, read files, and exit cleanly before the workshop starts.

2 · Sandbox

Create a disposable practice repo.

Use a toy app, copied template, or fictional notes folder. Keep it separate from production systems and private client work.

3 · Rules

Add a small instruction file.

Prepare a starter CLAUDE.md with rules like “ask before guessing,” “prefer small diffs,” and “verify before done.”

4 · Sample

Sanitize one input example.

Replace names, credentials, customer facts, URLs, and internal identifiers with fictional equivalents before using them in prompts.

5 · Evidence

Know how “done” will be proven.

Pick one visible result: a passing check, rendered page, clean diff, exported review packet, or checklist that another person can inspect.

Copyable worksheet

Use this as your readiness note

My safe practice target

The folder or repo I will use is: [toy repo / sanitized notes / small non-production feature].

My off-limits boundaries

The agent must not touch: production systems, secrets, payments, legal/health data, external messages, private customer material, or unpublished internal resources.

My first task

The small reversible task I want to try is: [summarize notes / draft a PRD / add one UI card / review one file].

My verification signal

I will call the task done only when I can show: [test output / screenshot / rendered page / diff / exported packet].

If something is not ready

Use the workshop anyway

No repo ready?

Use the product brief generator and PRD starter. Planning workflows are valuable even without a codebase.

No clean sample?

Use the public sanitized scenario chain instead of improvising with private company material.

No install access?

Follow the live demo and focus on the harness map, project rules, and review habits you can bring back later.
