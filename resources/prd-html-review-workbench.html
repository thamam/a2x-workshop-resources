PRD to HTML review workbench.

Paste PRD Markdown, render it into a readable review surface, leave local in-memory comments, and export a feedback packet for your next model or OpenSpec handoff.

Privacy boundary: this page is static. It makes no network calls, has no backend, uses no analytics, and does not save comments to localStorage or disk. Keep real client material out of public demos.

1. Source PRD

Use plain Markdown headings. Optional OpenSpec-like sections such as Why, What changes, Impact, and Acceptance criteria are recognized as normal sections.

2. Review surface

Missing-section checks look for users, non-goals, constraints, acceptance criteria, and risks. Comments stay only in this browser session memory.

Load the fixture or paste a PRD, then render the review page.

3. Export feedback packet

Copy this Markdown into your next assistant turn or OpenSpec proposal review.

Prototype scope: public-safe static page, local in-memory comments only, no persistence, no publishing approval implied.
