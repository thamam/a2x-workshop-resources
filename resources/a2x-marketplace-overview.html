Reusable agent workflow catalog

A2X Marketplace overview.

A public-safe way to explain the marketplace idea before sending attendees to any live repository: package repeatable Claude Code workflows as curated plugins, skills, setup notes, and review gates.

Why this page exists

Keep the concept public while the source path is curated

The marketplace source can become useful attendee material, but direct setup docs and plugin internals should be reviewed before linking from the workshop hub. This page teaches the model: turn repeated high-value workflows into installable, inspectable building blocks.

Block 1

Skills

Procedural memory for the agent: when to use a workflow, exact steps, verification commands, and pitfalls that should not be rediscovered.

Block 2

Plugins

Optional tool bundles that give Claude Code or a similar harness access to domain actions, structured commands, or external integrations.

Block 3

Profiles and presets

Opinionated combinations of rules, skills, tools, and safety defaults for a specific workshop, team, or workflow category.

Fictional marketplace card

Name: PRD Review Loop
Category: Product engineering
Helps with: converting a rough PRD into reviewable implementation tasks
Includes:
- skill: PRD critique checklist
- template: acceptance-criteria table
- command: export review packet
Safety notes:
- no customer data in examples
- no network calls by default
- requires human review before implementation

How to evaluate a marketplace item

  • Purpose: one sentence saying what repeatable pain it removes.
  • Scope: what the item can and cannot do.
  • Inputs: whether it expects files, prompts, browser access, credentials, or external services.
  • Verification: how a user knows it worked, including tests, previews, or review artifacts.
  • Safety: what data must not be pasted, uploaded, committed, or shared.

Workshop-safe installation framing

Do not ask a room of attendees to install an uncurated plugin bundle live. For the workshop, prefer one of these modes:

  1. Concept-only: show the fictional card above and discuss how skills/plugins reduce repeated agent setup.
  2. Facilitator demo: Tomer runs a prepared local environment and narrates the workflow while attendees watch.
  3. Curated handout: after source docs are cleaned, publish a minimal install path with placeholders such as [TOKEN], [API_KEY], and <plugin-dir>.

Red flags before public linking

  • Absolute local paths or personal machine assumptions in setup docs.
  • Credential, token, or API-key examples that look copy-pasteable.
  • Bundled code that creates secret-scanner noise and has not been explained or pruned.
  • Maintainer contact details or personal workflow notes that are not intended as attendee-facing support channels.
  • Claims that imply official support, pricing, reliability, or production readiness without review.
Static public-safe overview. No network calls, tracking, storage, direct source links, local paths, personal contact details, plugin internals, private notes, or real credentials.
