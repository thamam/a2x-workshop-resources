Pick the next AI tool trial by evidence, not hype.

Use this worksheet after the workshop when a team has several AI tools or workflows competing for attention. The goal is not to crown a universal winner; it is to choose one small, safe trial with a clear decision signal.

Safe evaluation rule

Score a trial only against a specific workflow.

Do not compare vague tool brands. Compare one tool or agent setup against one repeatable job: draft a spec, review a PR, summarize notes, build a prototype, triage support themes, or prepare a handoff packet. Keep private data, credentials, customer files, billing screens, and internal links out of shared scoring notes.

Step 1

Define the trial in one sentence

Workflow

What repeated job are we testing?

Example: “Turn a sanitized product idea into a reviewable PRD and OpenSpec proposal.”

User

Who must benefit?

Name the role: founder, product lead, engineer, operations owner, sales lead, or facilitator.

Evidence

What output proves value?

Choose a visible artifact: clean diff, review packet, accepted outline, saved time estimate, fewer missing questions, or better handoff.

Boundary

What is off-limits?

List anything the trial must not touch: production data, external messages, payments, secrets, private repos, customer material, or irreversible actions.

Step 2

Score the candidate

Use a 1–5 scale. A strong trial does not need perfect scores; it needs a clear reason to run now and a safe way to stop.

1. Workflow fit

Does the tool match a real repeated job, or is the team inventing a demo just because the tool is interesting?

1 vague2 weak3 plausible4 strong5 repeated pain

2. Safety and reversibility

Can the trial run on sanitized inputs, toy repos, copied docs, or read-only material with clear stop conditions?

1 risky2 brittle3 guarded4 safe5 reversible

3. Setup friction

Can the intended users start without new procurement, unclear credentials, private installs, or fragile environment assumptions?

1 blocked2 heavy3 manageable4 light5 ready today

4. Verification signal

Will the team know whether the trial worked from observable evidence instead of vibes?

1 vibes2 anecdote3 visible4 measurable5 decision-ready

5. Adoption path

If the trial works, is there a realistic next step: a rule file, reusable skill, checklist, workflow owner, or review habit?

1 orphan2 unclear3 owner maybe4 next step5 repeatable system
Step 3

Make the decision

Run it now

Total score is 18+ and safety/reversibility is 4 or 5. Define a tiny trial, use sanitized inputs, and package the next agent run with the handoff packet.

Clarify first

Total score is 13–17, or the verification signal is weak. Use the product brief generator, OpenSpec interviewer, or post-workshop action plan before running the tool.

Park it

Total score is 12 or below, or the workflow needs private data, production access, pricing commitments, or approval before safe evaluation.

Copyable trial note

Paste this into your planning doc

Candidate: [tool or workflow name]

Workflow: [one repeated job]

Safe input: [toy repo / sanitized sample / copied non-sensitive docs]

Off-limits: secrets, customer data, production systems, private links, external sends, payments, and irreversible actions.

Success evidence: [artifact, check, review packet, or before/after comparison]

Decision: run now / clarify first / park.
