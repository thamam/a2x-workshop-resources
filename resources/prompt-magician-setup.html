Prompt Magician setup overview.

Use this as the workshop-safe framing before anyone installs or configures a richer prompt-enhancement extension. The goal is better prompts, not extra setup friction.

Safety-first framing

What to say before demoing it

Prompt Magician is optional. Attendees can get the core workshop value from the Prompt improver handout alone. Use the extension path only when the room is ready to discuss browser/VS Code permissions, shortcut conflicts, and what text may be sent to model providers.

Mode 1

Local/offline first

Start with rule-based prompt cleanup, scoring, and rewrite structure where available. This keeps the exercise focused on prompt craft and avoids API-key setup during the workshop.

Mode 2

Remote LLM assisted

Only enable provider-backed enhancement after explaining that selected prompt text may be sent to the chosen model provider. Use placeholders like [API_KEY]; never paste real keys into shared docs or slides.

Mode 3

Editor workflow

For coding sessions, prefer an editor-based flow: select a rough prompt, request an improvement, inspect the rewrite, then save the pattern as a reusable project rule or skill.

Curated workshop quick start

  1. Use the simple Prompt improver page first so everyone understands the workflow.
  2. If demoing the extension, point attendees only to the curated Chrome folder or VS Code path prepared by the facilitator; do not send them browsing through all source docs live.
  3. Prefer local/offline enhancement for the first pass.
  4. If remote enhancement is enabled, configure provider keys outside the browser page and use [API_KEY] placeholders in any shared instructions.
  5. Test shortcuts before the room uses them. If defaults conflict, use alternatives such as Alt+Shift+E, Alt+Shift+Z, Alt+Shift+I, and Alt+Shift+C.

Permission and privacy talking points

  • Browser extension scope: broad page access can be normal for a text-selection extension, but attendees should understand what a browser extension can inspect before installing it.
  • Provider calls: model-backed enhancement may send selected text to an external provider. Do not test with private customer notes, secrets, internal strategy, or unpublished business details.
  • Workshop defaults: fictional examples are enough. The synthetic meeting-prep scenario is safer than live client material.
  • Audit status: this page is a sanitized overview, not a direct source-repo endorsement. Direct repository linking should wait for a curated release path.

Demo script in 90 seconds

  1. Paste a rough prompt: “Make my meeting prep better.”
  2. Ask for a rewrite with goal, context, constraints, output format, and quality bar.
  3. Compare the before/after prompt and name the improvement pattern.
  4. Save the pattern into a reusable prompt, rule, or skill rather than treating the improved prompt as a one-off trick.
