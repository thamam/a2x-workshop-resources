Facilitation rule

Name the moment before collecting details.

Before asking for examples, decide whether the room needs a concept, a safe demo, a discovery question, a parking-lot entry, a policy boundary, or a stop. Do not collect customer names, employee records, production logs, account screenshots, contracts, private repository links, raw prompts, confidential strategy, credentials, regulated data, or pricing details.

Live routing

Pick the next move in under thirty seconds

Teach

The room lacks a model

Explain the agent harness, project rules, review gates, or safe-input boundary before asking anyone to share examples.

Demo

A safe example is ready

Use a toy repo, fictional sample, or public-safe handout. Say out loud what would be off-limits in a real workflow.

Discover

The idea is vague

Ask for the repeated workflow, user role, allowed sample category, review owner, and smallest reversible next action.

Park

The question is useful but not ready

Capture the generic pattern, reason it is parked, owner role, safe evidence needed, and follow-up promise.

Escalate

A boundary is unclear

Route to security, legal, procurement, policy, sponsor, or workflow owner review without collecting sensitive examples.

Stop

The next detail would be unsafe

Pause the thread when the answer requires secrets, private data, production impact, regulated records, or unapproved tool use.

Resource handoffs

Use the cue to keep momentum

If the room wants examples

  • Use the readiness checklist or harness map.
  • Swap real details for toy or fictional samples.
  • State the stop condition before the demo.

If the room wants a decision

  • Open the prioritization matrix or use-case intake brief.
  • Name one owner role and one reviewer role.
  • Require evidence before rollout language.

If the room wants follow-up

  • Open the parking lot or follow-up email template.
  • Include only public-safe hub links.
  • Ask for one reversible next action.
