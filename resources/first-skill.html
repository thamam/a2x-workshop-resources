A2X workshop

Build your first skill.

A skill is a reusable instruction packet for a workflow you expect the agent to repeat. The trick is to save the process, not the outcome.

1. Pick a repeated workflow

Good candidates: running a project’s tests, preparing a pull request, reviewing a design, publishing a release, or debugging a known system.

2. Write the trigger

Say when the agent should load the skill. Be concrete.

Use this skill when working on release preparation for this repository.

3. Write the steps

Use commands, paths, checks, and pitfalls. Avoid vague advice.

1. Run the release checklist in docs/release.md.
2. Run tests with the project command.
3. Inspect git diff before committing.
4. Do not publish without approval.

4. Add verification

A good skill tells the agent how to prove the work is done.

Completion requires a clean git status, passing checks, and a link to the created artifact.

Workshop move

Ask the agent to perform a task once. Then ask it to turn the successful workflow into a skill. Run the same task again and compare.
