A2X field kit · output review

Review the AI output before anyone relies on it.

Use this one-page rubric when Claude, Claude Code, ChatGPT, an internal assistant, or a vendor tool produces a draft, summary, analysis, code change, recommendation, or workflow artifact. It helps reviewers catch confident-but-wrong output before it becomes operational reality.

Review rule

If the output changes a decision, workflow, customer message, system, or budget, it needs human review evidence first.

Review against visible criteria, source material, tests, policy boundaries, and reversible next steps. Do not approve output because it sounds polished. Keep secrets, customer data, employee records, production logs, account screenshots, private repository contents, credentials, regulated records, vendor contracts, pricing claims, and internal strategy out of shared review notes unless the approved environment allows them.

Step 1

Run the six-point output review

1. Task fit

Pass: the output answers the exact workflow question, acceptance criteria, or draft request. Watch: it solves a broader, adjacent, or more ambitious problem than requested.

2. Source support

Pass: claims can be traced to provided sources, visible evidence, tests, or sanitized notes. Watch: it invents facts, policies, timelines, product behavior, citations, numbers, or customer context.

3. Safety boundary

Pass: it respects allowed inputs, off-limits data, policy boundaries, and stop conditions. Watch: it asks for sensitive data, recommends private exposure, or assumes approval that has not been granted.

4. Operational clarity

Pass: the next step has an owner, review gate, timing, and success evidence. Watch: it leaves vague verbs like “coordinate,” “improve,” “monitor,” or “scale” without a concrete check.

5. Tone and promise control

Pass: language is accurate, modest, and appropriate for the audience. Watch: it overpromises savings, speed, accuracy, autonomy, compliance, pricing, or production readiness.

6. Reversibility

Pass: the proposed action is draft-only, local, sandboxed, staged, or approval-gated. Watch: it jumps to public publishing, external sending, production writes, broad rollout, or irreversible commitments.

Step 2

Choose the review decision

Approve for next safe step

All six review points pass. Record the evidence and proceed only to the bounded next action named in the output.

Revise before use

One or two points are weak. Send a correction prompt, add missing source notes, narrow the scope, or rewrite the output manually before relying on it.

Stop and escalate

The output conflicts with policy, lacks evidence for a consequential decision, asks for sensitive material, or proposes an irreversible action. Pause and route to the right owner or reviewer.

Copyable review note

Paste this under the AI output

Output reviewed: [draft / summary / recommendation / code change / workflow artifact].

Evidence checked: [source notes, acceptance criteria, tests, sanitized sample, policy boundary, reviewer judgment].

Pass / revise / stop: [decision] because [one-sentence reason].

Required changes: [missing source, narrower claim, safer boundary, clearer owner, review gate, rollback path].

Allowed next step: [draft-only / local test / reviewer approval / small pilot / do not proceed].

Do not include: secrets, customer data, employee records, production logs, account screenshots, private repository contents, credentials, regulated records, vendor contracts, pricing claims, or internal strategy in this shared note.
