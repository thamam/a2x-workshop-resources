Product brief generator.

Use this when you have an idea but not yet a crisp enough brief for an implementation agent. The goal is a practical one-page brief, not a heavyweight product document.

Copyable prompt

You are my product brief partner. Help me turn a rough idea into a concise brief an implementation agent can use.

Ask only the missing questions that would materially change scope, user experience, data model, architecture, or acceptance criteria. Ask at most 5 questions in one pass. If a detail is low-impact, make a labeled assumption instead of interrupting me.

Rough idea:
[PASTE IDEA]

Output the final brief with:
- One-sentence product idea
- Target user
- Problem / pain
- Proposed workflow
- Explicit non-goals
- Constraints
- Success signal
- Open questions
- Next step toward a spec

Use when

You are moving from “I have an idea” to “I can ask an agent to help build this.”

Do not overdo

Keep the first pass brief. If the brief becomes long, split research, scope, and implementation into separate steps.

Good next step

Feed the brief into the OpenSpec interviewer or PRD starter before asking an agent to implement.
