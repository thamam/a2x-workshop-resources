Safe prompt rule

If you would not put it in a workshop handout, do not paste it into a shared AI prompt.

Describe the task, role, decision, and acceptance criteria. Replace sensitive examples with fictional or sanitized samples. Keep secrets, customer data, employee records, production logs, contracts, account screenshots, private repository contents, credentials, regulated records, and internal strategy out unless the tool, policy, and approval boundary explicitly allow them.

Step 1

Run the five-gate preflight

1. Purpose is narrow

Safe: the prompt asks for one draft, explanation, checklist, diff, route, or decision aid. Unsafe: the prompt asks the model to decide policy, spend money, message customers, change production, or act without review.

2. Context is sanitized

Safe: examples use fictional customers, toy repos, redacted snippets, or copied non-sensitive docs. Unsafe: raw tickets, contracts, logs, screenshots, private links, credentials, employee/customer records, or confidential strategy are present.

3. Boundaries are explicit

Safe: the prompt names what the AI may use, what it must ignore, and what requires human approval. Unsafe: the prompt says “do whatever is needed” without stop conditions.

4. Output is reviewable

Safe: a human can check the answer against visible criteria, source notes, tests, or a simple pass/fail list. Unsafe: the answer would be accepted because it sounds confident.

5. Next action is reversible

Safe: the next step is draft-only, local, sandboxed, or approval-gated. Unsafe: the result could trigger external sends, production writes, public claims, pricing promises, or irreversible changes.

Step 2

Choose the prompt state

Send: all five gates pass

The prompt is narrow, sanitized, bounded, reviewable, and reversible. Send it, then review the output before acting.

Revise: one gate is weak

Rewrite the prompt before sending. Shrink scope, replace examples, add stop conditions, or define review evidence.

Stop: sensitive material is required

Do not paste the prompt into a general tool. Move to a policy-approved environment, use the pilot data boundary checklist, or ask for explicit approval.

Copyable prompt preflight note

Paste this above the prompt

Task: Help with [one narrow draft/checklist/diff/decision aid].

Safe context: Use only [fictional sample / redacted note / toy repo / public doc excerpt].

Do not use: secrets, customer data, employee records, production logs, account screenshots, private repository contents, contracts, regulated records, pricing claims, or internal strategy.

Boundary: Produce a draft for human review only. Do not send messages, change systems, make public claims, or take irreversible actions.

Review criteria: I will check [accuracy, source notes, tests, acceptance criteria, risk list, or pass/fail checklist] before acting.

If the prompt needs sensitive information: stop and say what approval or safer environment is required.
