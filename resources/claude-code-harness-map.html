A2X workshop

The harness, not just the model.

Claude Code gets dramatically better when the environment around it is deliberate: rules, tools, context, feedback, and permission boundaries. Use this map to explain what to improve before blaming the model.

Core idea

Agents perform inside a system

A coding agent is not only a chat box with a better model. It is a loop: inspect files, choose actions, edit, run tools, observe results, and recover. The workshop goal is to make that loop safer and more useful.

The five harness layers

1. Repository rules

Project instructions such as CLAUDE.md tell the agent how to behave before it touches code.

2. Navigable context

Readable file names, small docs, architecture notes, and examples give the agent handles it can follow.

3. Tool access

Tests, linters, search, formatters, local servers, and debuggers turn guesses into checked work.

4. Reusable workflows

Skills, slash commands, scripts, and checklists preserve workflows that worked once so they can work again.

5. Review gates

Diff review, secret scanning, approval boundaries, CI, and rollback plans keep autonomy from becoming recklessness.

Workshop diagnostic

When the agent struggles, ask which layer failed:

Did it know the rules?
Could it find the right context?
Could it run the right tools?
Was the workflow reusable?
Was there a verification gate?

Attendee action

Pick one active project. Add one repository rule, one verification command, and one reusable workflow note. Then run the agent on a small task and compare the quality of the diff.
