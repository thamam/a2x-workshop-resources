Library rule

A prompt library is a memory aid, not a place for secrets.

Save reusable structure, examples made from toy or fictional data, review criteria, and owner notes. Do not save customer records, employee data, production logs, account screenshots, credentials, regulated data, vendor contracts, pricing, private repository links, raw incident transcripts, or confidential strategy.

Prompt-card ingredients

Six fields worth keeping

Purpose

What job does it do?

  • Name the repeated workflow moment.
  • Say who should use it and when.
  • Include the decision it supports.
Safe input

What can be pasted?

  • List allowed input categories.
  • Use placeholders for sensitive details.
  • Point to the data boundary if approval is needed.
Prompt text

What should stay stable?

  • Keep the instruction short and narrow.
  • Separate variables from fixed instructions.
  • Write stop conditions into the prompt.
Review rule

How is output judged?

  • Name pass, revise, and stop criteria.
  • Link the output review rubric when helpful.
  • Require source support for risky claims.
Owner

Who maintains it?

  • Name a role, not a private person.
  • Set the review cadence.
  • Record where feedback should go.
Retire trigger

When should it change?

  • Mark tool, policy, workflow, or quality drift.
  • Pause if the prompt encourages unsafe inputs.
  • Archive cards no one reviews.
How to use it

Run a twenty-minute library pass

Pick one proven prompt

Choose a prompt that helped during the workshop, a pilot review, a meeting follow-up, or a low-risk workflow trial. If it has not worked at least once with safe input, keep testing before adding it to the library.

Strip out private context

Replace real names, customer details, private systems, account paths, source links, and internal examples with placeholders or fictional examples.

Add review and stop rules

Write how a human reviewer should accept, revise, escalate, or retire the prompt. A reusable prompt without review rules becomes folklore, little mouse.

Choose a storage boundary

Keep the prompt card in an approved internal location or team doc. This public handout does not define where your organization should store sensitive workflow material.

Good next moves

Where this prompt card fits

Before a skill

Use the card to decide whether the repeated pattern deserves a Claude Code skill, checklist, or workflow doc.

Before sharing

Run the prompt through the safety preflight and output rubric so reuse does not spread weak or unsafe instructions.

Before governance review

Bring cards with unclear inputs, ownership, or quality evidence into the intake brief, owner card, or policy exception request.
