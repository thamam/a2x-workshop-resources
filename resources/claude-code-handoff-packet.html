Safe scope

A good handoff lets the agent start small and stop safely.

Do not paste secrets, real customer material, production credentials, billing pages, private repository links, or account screenshots into a shared packet. Use fictional or sanitized examples until the workflow has a reviewed permission path.

Packet contents

Five things to capture before the next run

1 · Goal

Name the one outcome.

Write the smallest useful artifact: a draft spec, a testable page, a cleaned prompt, a review packet, or a toy workflow.

2 · Context

Point to the safe inputs.

List public docs, sanitized snippets, toy repo paths, or fictional examples. If the input is sensitive, replace it before the agent sees it.

3 · Rules

Set behavior and stop conditions.

Use a tiny CLAUDE.md or prompt preamble: ask before guessing, prefer small diffs, avoid production, and verify before declaring done.

4 · Next action

Ask for the first reversible move.

Do not begin with “build the whole thing.” Start with a plan, fixture, dry run, local page, failing test, or reviewable draft.

5 · Evidence

Define done before execution.

Choose the proof: command output, link check, browser smoke, diff review, exported packet, checklist, or a copied result.

Boundary

Record what is not allowed yet.

Examples: no external messages, no real credentials, no payment actions, no production deploys, no private-data processing.

Copyable template

Paste this into your notes or next Claude Code session

Handoff packet

  • Outcome: The next useful artifact is…
  • Current state: We already have…
  • Safe inputs: Use only these fictional/sanitized/public materials…
  • Project rules: Keep changes small, explain assumptions, and ask before touching anything outside the stated files or notes.
  • Stop conditions: Stop before production, secrets, money, customer data, external messages, legal/health data, or irreversible actions.
  • First reversible move: Start by…
  • Verification: We will know it worked when…
  • Handoff after completion: Return changed files, commands run, evidence, risks, and the next safest action.

Prompt to run

“Read this handoff packet. Before implementing, identify missing context, risky assumptions, and any approval boundaries. Then propose the smallest reversible first step and the exact verification command or inspection that will prove it worked. If the scope is safe, proceed; if not, stop and explain the blocker.”
