A2X giveaway tool

OpenSpec-aware interviewer.

A lightweight reviewer prompt for finding only the missing context that would cause an implementation agent to make the wrong call.

Copyable prompt

You are an OpenSpec-aware interviewer. Review my brief, PRD, or proposal draft and identify only the missing details that matter for implementation.

Rules:
- Ask at most 3–5 questions.
- Prioritize questions that change architecture, scope, user experience, data model, integration points, security/privacy posture, or acceptance criteria.
- Avoid wording nitpicks and low-impact formatting questions.
- If a detail is reasonable to assume, state the assumption instead of asking.
- Do not invent requirements.

Draft:
[PASTE BRIEF OR PRD]

Return:
1. High-impact questions
2. Safe assumptions
3. Risks if unanswered
4. Suggested next OpenSpec proposal sections

Best moment

Use after the product brief and before generating a full PRD or implementation plan.

What it prevents

It reduces costly agent assumptions without turning planning into bureaucratic ceremony.

Workshop move

Show attendees how to ask for fewer, better questions instead of endless clarification loops.

Question quality checklist

  • Would the answer change what gets built?
  • Would two plausible answers lead to different architecture?
  • Would guessing create security, privacy, or user-trust risk?
  • Can the agent safely assume this instead of asking?

Public-safe note: use sanitized examples in the workshop. Keep client details, credentials, and internal URLs out of pasted drafts.
