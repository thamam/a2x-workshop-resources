CLAUDE.md & coding rules cheat sheet.

A small project instruction file can remove a surprising amount of agent chaos. Start with these four rules, then adapt them to your repo.

1. Think before changing files

Ask the agent to inspect the current state and name the plan before editing. This reduces random rewrites.

Before editing, inspect the relevant files and explain the smallest safe plan.

2. Keep it simple

Bias toward boring code, narrow changes, and fewer moving parts.

Prefer the simplest solution that satisfies the requirement. Avoid new dependencies unless they clearly pay for themselves.

3. Be surgical

Protect unrelated code. This is especially useful in large repos.

Change only the files needed for the task. Do not refactor adjacent code unless it is required.

4. Stay goal-driven

Define done before you start. Then require verification against that definition.

Before claiming completion, verify every explicit requirement with tests, commands, rendered output, or file inspection.

Workshop move

Paste the four rules into a fresh CLAUDE.md, run the same coding task before and after, and compare the diff quality.
